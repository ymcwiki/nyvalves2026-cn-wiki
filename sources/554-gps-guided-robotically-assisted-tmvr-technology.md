# GPS 引导机器人辅助 TMVR：技术描述与未来展望

## 核心要点

* 报告核心观点是：结构性心脏机器人不只是“机器人导管”，价值来自规划、影像导航、机器人控制和数据学习形成一个可复制 workflow。
* Medtronic 的机器人命题被概括为 Brain + Eyes + Hands：Brain 是数字生态，Eyes 是影像与导航，Hands 是机器人和器械。
* TMVR 和 TTVR 被视为真实临床应用场景，因为解剖在运动、个体差异大且难以可视化，需要标准化入路、目标对齐、释放深度和团队流程。
* 目标是把专家依赖型操作转向系统赋能：患者特异计划、整合可视化、机器人辅助控制，以及用数据改进下一例。
* 未来生态覆盖术前、术中、术后：定义解剖风险、规划器械与路径、早期识别患者、实时可视化、受控输送、确认位置、追踪恢复、从每例中学习。

### 机会大于一根机器人导管

从“今天依赖专家”转向“明天系统赋能”的对比框架。

该页主标题为“The opportunity is bigger than a robotic catheter”。结构性心脏领域过去依靠医生经验进步，下一步是让这种经验更可复制。今天的模式是 expert-dependent，医生用技能和经验补偿复杂性，问题包括可变解剖、有限可视化、手动导管控制和高认知负荷。明天的模式是 system-enabled，系统帮助专家表现更可复制，对应四项能力：planned，患者特异性策略；visualized，整合引导；controlled，机器人辅助；learned，数据改进下一例。

| 可变解剖 | Planned：患者特异性策略 |
| 有限可视化 | Visualized：整合引导 |
| 手动导管控制 | Controlled：机器人辅助 |
| 高认知负荷 | Learned：数据改进下一例 |

### Medtronic 机器人命题：Brain、Eyes、Hands

机器人控制只有连接规划、影像和导航时才有更大价值，三部分为 Brain、Eyes、Hands。

该页标题为“The Medtronic Robotics Thesis: Brain + Eyes + Hands”。副标题指出，机器人控制在连接到 planning、imaging 和 navigation 时才有价值。Brain 代表 Digital Ecosystem；Eyes 代表 Imaging and Navigation；Hands 代表 Robotics and Instruments。中间横条强调，机器人只是系统的一部分，价值来自 Eyes、Hands 和 Brain 共同工作。页脚进一步定义：Brain 包括 planning、simulation、data 和 learning；Hands 包括 robotic control 和 catheter stability；Eyes 包括 imaging、navigation 和 registration。

| 模块 | 页中定义 | 功能外延 |
| --- | --- | --- |
| Brain | Digital Ecosystem | 规划、仿真、数据、学习 |
| Eyes | Imaging and Navigation | 影像、导航、配准 |
| Hands | Robotics and Instruments | 机器人控制、导管稳定性 |

### 结构性心脏病是机器人真实临床场景

TMVR 和 TTVR 需要在运动、可变、难以可视化的解剖中做精确决策。

该页指出，TMVR 和 TTVR 需要在运动、可变且难以可视化的解剖中作出精确决策，因此是机器人技术的真实临床使用场景。机器人可帮助标准化四个方面：access path，即到达瓣膜的最安全路径；target alignment，即器械与瓣环的方向关系；deployment depth，即关键时刻的释放深度精度；team workflow，即把专家步骤变成可教学流程。底部目标为“make excellent execution more reproducible across teams”，即让优秀执行在不同团队中更可复制。图中还展示了导管/器械组件、critical deployment handle、robotics cradle 等硬件图像。

| 机器人可标准化项目 | 具体含义 |
| --- | --- |
| Access path | 通向瓣膜的最安全路径 |
| Target alignment | 器械与瓣环方向对齐 |
| Deployment depth | 关键释放时刻的深度精度 |
| Team workflow | 将专家步骤做成可教学流程 |
