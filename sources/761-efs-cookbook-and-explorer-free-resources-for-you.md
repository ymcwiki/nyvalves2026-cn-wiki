# EFS Cookbook 与 Explorer：旅程中的免费资源

## 核心要点

* EFS 早期可行性研究的痛点不应由单个机构各自重复解决；MDIC、Heart & Valve Collaboratory 与 Veranex 试图把共同问题转化为共同工具。
* Cookbook/Playbook & Toolkit 被定位为免费、可持续更新的实用文件，用于减少策略、FDA 沟通、IDE 准备、预算、合同和中心启动中的摩擦。
* Explorer 是免费、实时的 EFS 地图工具，目标是把分散的公开数据集中呈现，帮助申办方理解中心、适应证和区域经验。
* Renata Medical 的案例显示，使用模板、预算工具、单次 CTA 和 Explorer 预筛 EFS 活跃心脏中心，可避免约 35,000 美元 的 CRO 启动费用。
* 讲者强调，EFS 的核心是“理解器械”，而 pivotal 研究才是“证明你认为自己已经知道的事情”。

### 为什么这项工作属于协作共同体

MDIC、H&VC 与 Veranex 分别提供中立平台、心脏瓣膜共同体和落地支持，合力把 EFS 痛点变成共享工具。

本页标题为“Why this work belongs in a collaborative community”。核心句为：当理念相近的人把共同痛点转化为共同工具时，EFS 会得到改善。

三方角色分别为：MDIC convenes the field，意为 MDIC 汇聚领域相关方；它提供一个中立场所，让申办方、FDA、临床医生、研究中心、患者和合作伙伴围绕实际解决方案对齐，这些方案不是单一机构能够独立完成的。H&VC brings the heart-valve community，意为 H&VC 带来心脏瓣膜共同体；这是以患者为中心、多学科协作的共同体，聚焦降低障碍并加速心脏瓣膜疾病创新。Veranex supports translation to practice，意为 Veranex 支持从概念转化到实践；其运营、临床、监管和证据生成经验帮助把概念变成团队真正能用的工具。

底部总结为：三方合力使 EFS 工作不只是一次资源发布，而成为一种共享操作系统，用于让早期临床评价更快、更清楚、更高效。

### Swallow Cliffs 的患者叙事

Swallow Cliffs, Illinois 的台阶和眺望照片，用个人经历说明患者愿意为更多有质量的时间承担新器械风险。

本页地点为 Illinois 的 Swallow Cliffs。讲者把这里称为“我父亲最后一次登上任何意义上的顶点”。左侧照片展示通往高处的长台阶，中间小图标注“Halfway up, looking out”，意为“走到半路，向外眺望”。

讲者写道，那一天两名下班后的消防员帮她父亲回到车上，而父亲“笑得很开心”。她进一步说：如果一种新器械能换来像那样的额外 30 天，她父亲会愿意承担风险。

### 共同体本身就是工作

从 Swallow Cliffs 的互助经验，过渡到 MDIC、H&VC、Veranex 与 EFS 社群的协作框架。

本页标题为“The Consortium Is the Work”。副标题说明：MDIC 与 H&VC 创造桌面，EFS 社群把共同障碍转化为共同解决方案。

左侧“来自 Swallow Cliffs 的经验”写道：最好的共同体让艰难攀登成为可能。在 Swallow Cliffs，陌生人会成为盟友，人们推动、鼓励，并在有人需要帮助时出现。这正是 MDIC 与 H&VC 背后的精神：把正确的人带到一起，围绕共同目标，让等待创新的患者能够看到进展。

右侧“MDIC + H&VC + Veranex”列出三点。第一，利益相关方对齐：申办方、FDA、临床医生、研究中心、CRO、患者和合作伙伴拥有不同角色，但目标一致。第二，共同障碍转化为共同工具：Playbook 与 Explorer 把反复出现的痛点变成实用起点。第三，用行动带头：更快启动、更清楚的预期、更少返工。

### Cookbook：由共同体构建

本页标题为“The Cookbook: Built by the Community”。它被描述为免费、动态更新的文件，设计目的为减少摩擦、澄清预期，并加速负责任的 EFS 执行。

Cookbook 的四项功能为：第一，clarify the path，厘清路径，减少从早期策略、FDA 互动到 IDE 准备过程中的不确定性。第二，reduce startup friction，降低启动摩擦，为申办方和研究中心在可行性、预算和合同方面提供共同起点。第三，improve efficiency，提高效率，支持更快决策、减少重复问题，并让研究启动更一致。第四，keep it current，保持更新，通过反馈帮助 MDIC 与 H&VC 改进工具包，使它反映团队真实需要。页底列出资源名称：EFS Playbook & Toolkit。

### Explorer：为现场构建的地图

Explorer 把分散公开数据整合为 EFS 地图，并邀请用户从可用性、完整性、缺口和准确性四方面反馈。

本页标题为“The Explorer: The Map Built for the Field”。Explorer 被描述为免费、实时的工具，设计目的为支持更聪明的规划和信息更充分的研究中心选择。

对 EFS 社群的价值包括：把分散的公开数据整合起来，让团队能在一个地方查看 EFS 生态；让申办方、研究中心和治疗领域活动更容易理解；支持战略性中心选择和区域规划；帮助生态系统理解经验在哪里存在、缺口在哪里仍然存在。

讲者请求使用者帮助改进地图，反馈维度包括：可用性，界面是否清楚且易导航；完整性，数据字段和筛选器是否正确、足够；缺口，缺少哪些功能、指标或地理区域；准确性，若底层数据看起来不对，请标出。
