# AI 辅助结构性心脏介入超声引导（DeviceGuide）：临床经验与未来展望

## 核心要点

* DeviceGuide 被定位为介入超声的第一代 AI 工具，目标是在结构性心脏操作中把部分 MPR 引导步骤半自动化。
* 与传统 echo-fluoro fusion 相比，DeviceGuide 的核心差异是把器械轨迹和尖端在 echo、3D 与透视视图中持续显示，并可随探头移动进行跟踪。
* “Target” mode 由操作者设定 MPR 目标；黄线代表 trajectory，蓝点代表 tip，并提供 intercommissural、grasp、device view 和 3D orientation。
* “Device” mode 则让长轴、短轴和 3D 视图保持锁定在器械上，用于持续追踪器械位置。
* 病例演示围绕 PASCAL Ace 相关操作，逐步比较标准 MPR、融合轨迹和 DeviceGuide 在 steering、trajectory、grasp、deployment 中的显示方式。
* 讲者总结：DeviceGuide 仍依赖基础 MPR workflow，器械跟踪效果较好；未来改进方向包括工作流、运动平滑和支持更多器械。

### 传统 echo-fluoro fusion：超声与透视并列对照

传统 echo-fluoro fusion：左侧为 TEE 双平面图像，右侧为融合后的透视图，红框标出器械区域。

本页题为 “Traditional Echo-Fluoro Fusion”。画面左侧显示 TEE 的双平面超声图像，页面 UI 可见 X8-2t 探头、深度 10 cm、心率约 65 bpm 等信息。蓝色箭头指向右侧透视图，透视图叠加紫色解剖轮廓和绿色/粉色引导线，红色方框强调器械所在区域。该页说明传统融合主要是把超声平面和透视影像叠加，操作者仍需在不同视图间理解器械相对位置。

### DeviceGuide 的差异：在 echo 与 X-ray 之间同步器械导航

DeviceGuide 界面同时显示 Navigator、Echo 和 X-ray 视窗，蓝色双向箭头表示超声导航与透视显示的对应关系。

本页题为 “What’s Different with DeviceGuide”。左侧主界面上方标注 Navigator，中间为超声 MPR 大视图，左栏包含两个二维切面和一个 3D 视图；右侧为 X-ray 1 透视窗口。蓝色双向箭头位于超声和透视之间，表示两种模态之间的对应。底部蓝框中可见 PASCAL Ace 器械小窗和两个绿色勾选标记。页面传达的要点是：DeviceGuide 不只是静态融合影像，而是在超声 MPR 与透视视图之间持续提供器械相关的空间导航。

### Target mode：先设定 MPR 目标

Target mode 中，操作者设定 MPR target；黄线代表轨迹，蓝点代表器械尖端。

本页题为 “Target” Mode。左侧文字定义：Yellow line = trajectory，Blue dot = tip。中央图像写明 “You set this MPR ‘target’”。四个视图分别为 Intercommissural、Grasp、Device view (auto tracks) 和 3D for orientation。页面上可读到的角度包括 intercommissural 视图约 RAO 35°、CRAN 11°，grasp 视图约 RAO 62°、CAUD 71°，device view 约 RAO 75°、CAUD 78°，3D orientation 约 RAO 128°、CRAN 8°。右侧透视图叠加粉色目标轮廓和轨迹，底部红框标示 RAO 37°、CRAN 12°。

### 总结与未来方向

讲者总结：DeviceGuide 是介入超声的第一代 AI 工具；某些步骤变为半自动化；系统依赖底层 MPR workflow；器械跟踪运行良好，包括探头移动时也能工作。未来改进预计包括优化工作流、平滑运动，以及支持更多器械。页面最后显示 “Thank you!” 和联系邮箱 [邮箱已隐去]。
